# EssayMate Product Design

## User Problem

Junior high students often receive English writing feedback that is hard to act on.

They may know that an essay lost points, but they do not clearly know:

- which mistake matters most
- how to rewrite a sentence
- what phrases to remember
- how to improve in the next practice

EssayMate is built to turn correction into a learning loop.

## Target Users

Primary users:

- junior high English learners
- students preparing for exam-style writing
- learners who need simple, structured feedback

Secondary users:

- teachers who want a fast feedback demo
- parents who want to understand a child's writing weaknesses
- AI product interviewers evaluating product thinking

## Product Positioning

EssayMate is an AI Writing Coach for students.

It is not only a scoring tool.

It is a structured learning assistant that helps students move from:

Write -> Analyze -> Learn -> Retry -> Improve

## Core Flow

1. Write

Students paste an English essay and choose their writing level.

2. Analyze

The system provides a score, four-dimension evaluation, key problems and suggestions.

3. Learn

Students review sentence-level corrections, a model essay, key phrases and learning tips.

4. Retry

Students rewrite the essay or generate a similar topic for another practice.

5. Improve

The product shows progress and reinforces what the student learned today.

## Feature Design

### Essay Input

Simple form-based input:

- topic
- essay draft
- student level
- sample essay
- clear input

### AI Analysis

The feedback is divided into scannable modules:

- score
- content
- grammar
- structure
- vocabulary
- key problems
- suggestions

### Sentence-Level Correction

Each sentence shows:

- original sentence
- improved sentence
- reason

This helps students understand the change, not only copy the answer.

### Learning Feedback

The learning module includes:

- model essay
- key phrases
- learning tips
- what you learned today

### Learning Progress

The progress section shows:

- before feedback
- after learning
- today's improvement

This makes the product feel like a continuous education system instead of a one-time checker.
